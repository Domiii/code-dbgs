# The Run Button

import Img from '@src/components/Img';
import DarkLightImg from '@src/components/DarkLightImg';


The Run Button <DarkLightImg src='play.svg' /> allows the user to easily run a simple Node application [with Dbux enabled](./).

You can find it in the top right of opened `*.js` files:

<Img src="screens/run-button.png" className="border-screenshot" />

:::tip
The Run button simply calls the `Dbux: Run Current File` command. You can also access it via [VSCode's command palette](https://code.visualstudio.com/docs/getstarted/userinterface#_command-palette), and (like [all commands](../tools-and-configuration/01-dbux-code.mdx#commands)) can be key-bound.
:::

:::note
The `Dbux: Debug Current File` command does the same thing as the Run button but with `--inspect-brk` enabled.
:::

## Results

When pressed, a Terminal opens and starts executing the active `*.js` file with [@dbux/cli](../tools-and-configuration/02-dbux-cli.mdx), e.g.:

> ```$ node --stack-trace-limit=1000 --enable-source-maps -r ./my-code/node_modules/@dbux/cli/dist/linkOwnDependencies.js "./my-code/node_modules/@dbux/cli/bin/dbux.js" run --esnext --pw=.* "./my-code/firstDuplicate/find-bad.js"```
