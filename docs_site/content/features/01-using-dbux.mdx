Using Dbux in three steps:

1. [Install Dbux](#installation)
2. Make sure to [run an Application with Dbux enabled](./02-enable-dbux.mdx).
3. Use the <ToolLink name="dbux-code" /> for inspection + analysis.

All recorded applications show up in the [Application View](./05-applications.mdx).
Once the application shows up, you can use the following tools for analysis:

* Use the [Application View](./05-applications.mdx) to overview and toggle all recorded applications.
  * From here you can also go to the ***entry point***.
* Get a high-level overview with:
  * The [Global View](./08-global.mdx).
  * The [Call Graph](./10-call-graph.mdx) or [Asynchronous Call Graph](./20-acg.mdx).
* Get down into the nitty-gritty:
  * &#8203;<DarkLightImg src="crosshair_red.svg" /> [Select a Trace](./06-select-trace.mdx).
  * Investigate individual traces using the [Trace Details View](./07-trace-details.mdx).
  * &#8203;<DarkLightImg width="24px" src="nextInContext.svg" /> <DarkLightImg width="24px" src="previousInContext.svg" /> [Navigate](./07-trace-details.mdx#navigation) between traces.
* Understand where data came from and where it went using the [Data Flow View](./50-data-flow.mdx).
* [Search](./40-search.mdx) for traces, contexts, modules or data/variable/expression values.

:::tip
All buttons in the Dbux VSCode Extension have corresponding [commands](./tools-and-configuration/dbux-code#commands) which in turn (are not by default but) [can be key-bound](https://code.visualstudio.com/docs/getstarted/keybindings).
:::
